# Forge v0.1 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Merge the strongest parts of `feat/production-ready` and `feat/web-preview` into a final `v0.1` build where users can launch The Forge from one command, complete setup inside the app, auto-start the orchestrator, and optionally open a browser preview from the staging screen.

**Architecture:** Use `production-ready` as the integration base because it already contains setup, config, backend-mode, and release packaging work. Bring over the browser preview from `web-preview` as a staging-only enhancement inside `BubbleTeaTerminal`, keeping the TUI as the single control surface. Finish by rewriting docs and release/install flows around one canonical startup path.

**Tech Stack:** Go with Bubble Tea, Python 3 agents runtime, Git worktrees and branch merges/cherry-picks, GitHub Actions, shell install scripts, `go test`, `pytest`
